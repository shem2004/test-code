Push HTML to GitHub — A simple tutorial

Step-by-step guide that shows commands, a terminal preview, and an embedded live preview area.

1. Prepare your project

Create a folder and add your HTML, CSS, and assets. Example structure:
my-website/
├─ index.html
├─ style.css
└─ images/

2. Initialize Git & make your first commit

Run these commands in your project's root folder.
git init
git add .
git commit -m "Initial commit - add website files"

3. Create a remote repository on GitHub

Go to GitHub & create a new repo (public or private). Copy the repository URL.
# example remote URL — replace with your repo
git remote add origin https://github.com/yourusername/your-repo-name.git

4. Push your files to GitHub

Push to the default branch (main). If your repo uses master replace accordingly.
git branch -M main
git push -u origin main
Open live preview

5. (Optional) Host with GitHub Pages

To publish your site with GitHub Pages from the main branch:
# In repository settings: GitHub Pages -> Source -> branch: main / root
# Or use gh CLI:
gh repo edit --enable-pages

Tips

  • Use .gitignore for node_modules or build files.
  • If using SSH, use the SSH remote URL instead of HTTPS.
  • Enable GitHub Pages in repo settings to publish to the web.
